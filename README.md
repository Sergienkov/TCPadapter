# TCPadapter
Кафка tcp адаптер
Требования:
Функции адаптера	
1	Управление TCP‑соединениями с контроллерами
2	Обмен с Kafka: producer телеметрии/событий и consumer команд
3	Преобразование бинарного протокола контроллеров ↔ внутренний JSON
4	Получение данных с кафка шины и формирование буферов/очередей под каждый контроллер индивидуально
5	Контроль актуальности сообщение в буфере/очереди на предмет уникальности (не задвоения команд) и актуальности по времени (TTL)
6	Персистентное состояние буферов и сессий (в случае рестарта)
7	Exactly-once delivery режим доставки сообщений к контроллеру (через retained messages + idempotent processing)
8	Реализация механизма dead-letter queues для контроля и мониторинга ошибок
9	Компоновка сообщений в пакеты протокола если это не сделано уровнем выше, отработка ответов на сгруппированные сообщения (развитие например добавления идентификаторов по 20 идентификаторов в tcp пакет)
10	Приоритезация очередей команд для контроллера
	
Работа с контроллерами	
1	Модуль поднимает TCP‑сервер и принимает до 10 000 одновременных соединений от контроллеров.
2	Осуществляет распределение нагрузки 
3	Поддерживает протокол команду первичной идентификации
4	Реализует keepalive/heartbeat, осведомляет систему о смене статусов
5	Реализует корректное восстановление связи/сессий
6	Формирует очередь/буфер для каждого контроллера
7	Разделяет команды в очереди по приоритетам, определяет целесообразность отправки по времени жизни команды
8	Группировка / разгруппировка сообщений в пакеты при необходимости
9	Контроллирует уникальность (отсутвие дублей информации/команда) и актуальность команд по времени (TTL)
10	Обработка ошибок при отправке
11	Логирование всех событий для отладочного режима, нужно для полной отладки контроллеров.
	
Нормализация протокола	
1	Приём бинарных сообщений от контроллера
2	Отправка бинарных команд
3	Поддержка нескольких версий протокола
4	Поддержка приоритетных команд
	
Интеграция с Kafka	
1	Producer телеметрии/событий
2	Consumer команд
3	Логика TTL/удаление неактуальны команд/сообщений
4	Обработка ack
